JavaScript Execution Context

What is it?

An execution context is the environment where JavaScript code runs. It controls the flow of execution.

Types of Context

  • Global Execution Context (GEC) – Default context
  • Function Execution Context (FEC) – Created on function call
  • Eval Execution Context – Created by eval()

Phases

  • Creation Phase
    • Hoists variables/functions
    • Sets up this and scope chain
  • Execution Phase
    • Executes code line by line
    • Assigns values and runs logic

Call Stack

Execution contexts are pushed and popped from the Call Stack.

main() → foo() → bar()

Interview Tips

  • Explain GEC and FEC clearly
  • Mention hoisting during Creation Phase
  • Relate execution context to Call Stack
  • Show example of nested function calls
