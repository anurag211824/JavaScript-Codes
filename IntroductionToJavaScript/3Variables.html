JavaScript Variables

In JavaScript, variables are used to store data values. They can be declared using the keywords var, let, or const.

1. var

The var keyword was traditionally used to declare variables. Variables declared with var are function-scoped and can be redeclared and updated.

var x = 10;
console.log(x); // Output: 10

2. let

The let keyword is block-scoped and is recommended for most use cases. Variables declared with let cannot be redeclared but can be updated.

let y = 20;
console.log(y); // Output: 20

3. const

The const keyword is also block-scoped and is used for variables that should not be reassigned. The value of a const variable cannot be changed once it is set.

const z = 30;
console.log(z); // Output: 30

Variable Scope

Scope determines where variables can be accessed or modified in your code. JavaScript has three types of scope:

  • Global Scope: Variables declared outside any function are in the global scope.
  • Function Scope: Variables declared with var inside a function are scoped to that function.
  • Block Scope: Variables declared with let or const are scoped to the block in which they are defined.

Examples

// Global Scope
var globalVar = "I am global";

function testScope() {
    // Function Scope
    var functionVar = "I am function-scoped";

    if (true) {
        // Block Scope
        let blockVar = "I am block-scoped";
        const blockConst = "I am also block-scoped";
        console.log(blockVar); // Output: I am block-scoped
        console.log(blockConst); // Output: I am also block-scoped
    }

    console.log(functionVar); // Output: I am function-scoped
    // console.log(blockVar); // Error: blockVar is not defined
}

testScope();

Conclusion

Use let and const for better scoping and to avoid issues with redeclaration and hoisting. Reserve var for legacy code or specific cases where function scope is required.
